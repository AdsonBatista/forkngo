Fork-n-Go!

| fôrk en gō | adjective Forking a website repository on GitHub designed to be easily used by others.

Part I: GitHub Pages

GitHub Pages is free web hosting for files files on a branch named gh-pages in a repository. They can be found served up (like this ⤴︎ site) at: username.github.io/repository

Part II: Fork

Open source GitHub repositories can be forked by anyone else on GitHub. This creates a copy of that repository on your account. When you fork a website, you get a website.

Part III: Commit

Edit the index.html to make the changes you need to make it your own. Then commit (like saving) and your website will be live. You can do all this through github.com!

What's this all about?

GitHub's Pages and fork features are awesome tools that can bring the bar to getting started in web development way down.

Having something to start tinkering with and not worrying about hosting. Fork and get going! Change the CSS/HTML, tinker with JS — if the project is connected to a spreadsheet, interact with data!

Besides being a tool for tinerking-to-learn, it also makes having a live and hosted website more acessisble to those who may think it can't possibly be so easy.

I'd love to see Fork-n-Go pages that can be used for non profits and community groups. Tool lending libraries, directories of services...

Sheetsee.js Spreadsheet Fork-n-Go's

Fork-n-Go sites that use Sheetsee.js to connect the site to a spreadsheet make it easy to have your own website connected to your own data source.

If you create your own spreadsheet and copy the column headers from the original, you can then replace to old spreadsheet URL (or key) in the index.html of your fork and start using your own data.
